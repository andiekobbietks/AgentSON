# Changelog

All notable changes to AgentSON will be documented here. AgentSON uses [Semantic Versioning](https://semver.org/).

## [0.2.1] - 2026-07-08

### Added

- **`agentson validate` command** ‚Äî Validates .agentson files against v1.2 schema. Supports JSONL and single-document JSON. Entry-level validation via flattened schema.
- **70 schema validation tests** ‚Äî `tests/test_schema.py` covering all 12 primitives (valid + invalid cases), full document validation, edge cases, CLI integration, and provenance fields.
- **Flattened entry schema** ‚Äî `spec/v1.2-entries.json` (no $ref dependencies, CLI-ready validation). Generated by `tools/flatten_schema.py`.
- **ISO litmus test** ‚Äî Added to `spec/ontology.md` and all ADRs (015/016/017). Five-question readiness assessment for ISO standardization.
- **ADR-015: Canonical Schema** ‚Äî v1.2 JSONL is the canonical format. Deprecates v1/v1.1 single-document shape. **Accepted.**
- **ADR-016: Ontology‚ÜîEnum Reconciliation** ‚Äî 12 primitives are source of truth. Removed `context`/`querying`/`title`. Added `observation`/`user-feedback`/`system-event`/`capabilities`. **Accepted.**
- **ADR-017: Core vs Streaming Separation** ‚Äî Streaming primitives optional in single-agent, required in multi-agent. `mode` field in `stream-meta`. **Accepted.**

### Changed

- **`v1.2.json` rewritten** ‚Äî 12 primitives with real `required` fields (not empty), `oneOf` discriminated union, `mode` field in stream-meta, `heartbeat` field.
- **AGENTS.md updated** ‚Äî Schema status reflects v1.2 as canonical (was marked "Broken"). Dependency graph shows WP1-WP4 complete. File map updated with new files.
- **`wiki/ADRs.md` index complete** ‚Äî Added ADR-012, ADR-013, ADR-014. Updated ADR-015/016/017 status to "Accepted".
- **`pyproject.toml` dependencies** ‚Äî Added `openpyxl>=3.1` and `pylatex>=1.4` (were used but not declared).

### Removed

- **`cmd_push` and `cmd_pull`** from CLI ‚Äî Per ADR-013, push/pull extracted to downstream package. CLI now has 10 commands (was 12).
- **Dead code** ‚Äî Removed unreachable `if args.output:` block in `cmd_list` and duplicate `cmd_search` stub.

### Architecture

- v1.2 JSONL is now the only canonical format (ADR-015)
- 12-primitive ontology is source of truth for schema enum (ADR-016)
- Core vs streaming primitive separation with `mode` field (ADR-017)
- AgentSON positioned as "PDF of agent sessions" ‚Äî only provenance format in agent stack
- ISO readiness: 5-question litmus test in ontology.md and ADRs

## [0.2.0] - 2026-07-07

### Changed

- **Heartbeat visualizer rebuilt** using Google OKF design system (Cytoscape.js 3.28.1)
  - BEFORE: 43 lines, basic terminal-style, dark background (#0a0a0a), monospace font, flat SVG sparkline, pulsing animation
  - AFTER: 300+ lines, interactive knowledge graph, split panel (60% graph / 40% detail), light background (#f8fafc), system font, search with dim/non-dim, health filter, layout picker, backlinks, markdown rendering
  - Preserved original as `exports/heartbeat-BEFORE.html` for reference

### Added

- **AGENT_INDEX.md** ‚Äî 6.6KB compressed documentation index (Vercel approach). Agent reads this first, knows what exists, only fetches specific files when needed.
- **Heartbeat skill section** ‚Äî Visualizer rebuild process documented in `.opencode/skills/agentson-architect.md`
- **Compressed index skill section** ‚Äî Process for creating 8KB agent-readable indexes
- **Scope creep tracking** ‚Äî Inventory of 2,408 lines of unplanned work with prevention rules
- **OKF digest** ‚Äî `spec/okf-digest.md` (4.6KB) for offline OKF reference
- **Session timing R&D** ‚Äî `tools/session_timing.py` (burst detection, idle analysis, hydration, duration prediction)
- **Live heartbeat demo** ‚Äî `tools/live_heartbeat.py` (real-time terminal display)
- **Heart monitor** ‚Äî `tools/heart_monitor.py` (ASCII + HTML dashboard with SVG sparkline)

### Architecture

- Heartbeat field added to `spec/v1.json` (ISO8601, format: date-time)
- `readers/opencode.py` computes `heartbeat = max(entry.timestamp)`
- SQLite `last_heartbeat` column populated (137 rows)
- OKF ‚Üî AgentSON relationship clarified: OKF = knowledge layer, AgentSON = provenance layer
- First-mover strategy: build bridge (okf_adapter.py), don't compete with OKF

Patch release. Fixes bugs present in v0.1.0 that were claimed working but weren't.

### Fixed

- **File extension casing** (`.AgentSON` ‚Üí `.agentson`): every writer, reader,
  CLI export path, and search glob across `cli/main.py`, `readers/claude_code.py`,
  `importers/chatgpt.py`, and `tests/test_claude_code.py` used the brand-name
  casing instead of the lowercase extension decided in ADR-011. Fixed everywhere
  in one pass rather than piecemeal, since a partial fix (writer changed, glob
  not) would have silently broken file search on case-sensitive filesystems.
- **`importers/chatgpt.py` could not be imported at all**: a missing indent
  under a `with open(...)` block was a hard `IndentationError`, meaning
  `agentson import chatgpt` has not functioned since this file was committed,
  despite being listed as shipped in the v0.1.0 release notes below.
- **`importers/chatgpt.py` crashed on real ChatGPT exports**: `_detect_model`
  used `node.get("message", {})`, which only supplies the default for a
  *missing* key ‚Äî ChatGPT's root node has an explicit `"message": null`, so
  the call returned `None` and the next `.get()` raised `AttributeError`.
  This means the "validated against a real export, hidden system-injected
  profile data correctly excluded" claim in the v0.1.0 notes below did not
  reflect a working state of this file; there was also no test file for this
  importer prior to this fix. Added `tests/test_chatgpt_importer.py` with a
  fixture matching the real null-root-message export shape.
- **`readers/opencode.py` silently dropped `reasoning` parts and
  misread `tool` parts**: the reader was built against an older opencode
  part shape (`type: "tool-invocation"`, fields `toolName`/`args`/`result`).
  Verified against a real opencode message/parts export, current opencode
  emits `type: "reasoning"` (previously unhandled, entries silently lost)
  and `type: "tool"` with fields nested under `state` instead. Added
  `tests/fixtures/opencode_real_sample.json` (a real export, not synthetic)
  and `tests/test_opencode_real_sample.py`; old shape kept as a fallback
  for older opencode exports, not removed.

### Known issues (flagged, not yet fixed)

- `tests/test_claude_code.py::test_read_session` fails on `master`
  independent of the above: expects `outcome == "success"`, actual value
  is `"partial"`. Confirmed pre-existing (reproduces on `master` before
  any changes in this entry) and unrelated to the fixes above. Left open
  as a separate issue rather than folded into this fix, since the cause
  hasn't been diagnosed yet.

## [0.1.0] - 2026-07-05

Initial public release. Apache 2.0 license.

### Specification

- **JSON Schema v1** (`spec/v1.json`) ‚Äî 8 entry types: `user-query`, `context`, `querying`, `title`, `thought`, `action`, `answer`, `side-effect`
- **JSON Schema v1.1** (`spec/v1.1.json`) ‚Äî Trajectory semantics: `task`, `outcome`, `observation` entry type, `tool_call_id`, `correlation_id`, `status` on all entries, `duration_ms`, `error` on actions
- Backward compatible: all v1.0 files validate against v1.1

### Readers (6)

- `readers/opencode.py` ‚Äî opencode SQLite sessions
- `readers/minimax.py` ‚Äî MiniMax SQLite sessions
- `readers/antigravity.py` ‚Äî Antigravity IDE protobuf sessions
- `readers/libre.py` ‚Äî FreeStyle Libre 2 CSV glucose data
- `readers/chrome_devtools.py` ‚Äî Chrome DevTools AI session traces (9 tests)
- `readers/claude_code.py` ‚Äî Claude Code JSONL session transcripts

### Importers (1)

- `importers/chatgpt.py` ‚Äî ChatGPT `conversations.json` ‚Üí AgentSON

### Exporters (1)

- `exporters/finetune.py` ‚Äî Unsloth (ShareGPT) + Olive (chat-messages) training formats

### CLI

- `agentson export` ‚Äî Export sessions from any supported tool
- `agentson import` ‚Äî Import ChatGPT conversations
- `agentson list` ‚Äî List sessions with metadata
- `agentson search` ‚Äî Full-text search across sessions
- `agentson finetune` ‚Äî Export to training formats
- `agentson render` ‚Äî Render sessions as Markdown
- `agentson push` ‚Äî Push to Supabase (optional)
- `agentson pull` ‚Äî Pull from Supabase (optional)

### Web Viewer

- `docs/viewer/index.html` ‚Äî Full-featured: file extension + schema validation, download as .AgentSON, search/filter, sort toggle, copy per entry, side-by-side layout, role-based colors, collapsible JSON metadata preview
- `pwa/` ‚Äî Standalone PWA with offline support

### Documentation

- 14 SOPs (SOP-001 to SOP-014) covering glucose monitoring, AI session export, Nightscout, WhatsApp bot, TV dashboard, MedGemma, NHS FHIR, West Africa voice alerts
- Operating manual: mental models, coding standards, workflow rules, 10 ADRs
- Live docs at `https://andiekobbietks.github.io/AgentSON/`

### Examples

- `examples/opencode_example.AgentSON`
- `examples/minimax_example.AgentSON`
- `examples/antigravity_example.AgentSON`
- `examples/chrome_devtools_example.AgentSON`

### Infrastructure

- PyPI package: `pip install agentson`
- GitHub Actions CI: Pyrefly type checking, pytest
- Branch protection: `andierules` ruleset (PR required, linear history)
- CodeRabbit for AI code review (optional, not required)
